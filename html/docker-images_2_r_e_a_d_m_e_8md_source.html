10 ## Building Windows Docker Images and Uploading them to AWS Elastic Container Service
11 ### Prereqs
12 * Windows with Containers (Any Windows 10 install with the Anniversary update will do)
13 * Docker
14 * AWS CLI
15 
16 ### Building the Image
17 
18 Run build.ps1:
19 
20  powershell .\build.ps1
21 
22 After the build finishes, you can run and test the image by running:
23 
24  docker run -it vs2015
25  --or--
26  docker run -it vs2017
27 
28 To emulate the tests run by AWS CodeBuild, execute the following:
29 
30  git clone -b v1.0.0 --depth 1 https://github.com/aws/aws-encryption-sdk-c.git
31  cd aws-encryption-sdk-c\codebuild
32  .\codebuild\common-windows.bat
33 
34 When you are satisfied the image is to your liking, simply exit the container.
35 
36 ### Publishing the Image
37 
38 If you are publishing to your own account, update the `ECS_REPO` value in
39 `push.ps1`. You can find the correct URI in the AWS Console for your ECR
40 repository.
41 
42 Once you have `ECS_REPO` set properly, and you have configured your Powershell
43 AWS CLI credentials correctly, simply _source_ push.ps1:
44 
45  . .\push.ps1
46 
47 Note that because powershell CLI credentials are per-powershell-session, it's
48 important to use dot-sourcing if you use the `Set-AWSCredential` cmdlet to configure
49 your credentials. If you're using EC2 Instance Roles, then it's not strictly necessary
50 to dot-source the script.
