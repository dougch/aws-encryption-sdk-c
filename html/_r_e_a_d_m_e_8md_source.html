1 # AWS Encryption SDK for C
2 
3 The AWS Encryption SDK for C is a client-side encryption library designed to make it easy for
4 everyone to encrypt and decrypt data using industry standards and best practices. It uses a
5 data format compatible with the AWS Encryption SDKs in other languages. For more information on
6 the AWS Encryption SDKs in all languages, see the [Developer Guide](https://docs.aws.amazon.com/encryption-sdk/latest/developer-guide/introduction.html).
7 
8 Also, see the [API documentation](https://aws.github.io/aws-encryption-sdk-c/html/) for the AWS Encryption SDK for C.
9 
10 ## Dependencies
11 
12 The only direct dependencies of this code are OpenSSL 1.0.2 or higher or 1.1.0 or higher and
13 [aws-c-common](https://github.com/awslabs/aws-c-common) v0.4.15. You will also need
14 a C compiler and CMake 3.9 or higher.
15 
16 To integrate with [KMS](https://aws.amazon.com/kms/) the AWS Encryption SDK for C also requires
17 the [AWS SDK for C++](https://github.com/aws/aws-sdk-cpp), a C++ compiler, and libcurl.
18 
19 For best results when doing a build with KMS integration, do not install aws-c-common directly.
20 Build and install the AWS SDK for C++, which will build and install aws-c-common for you (see the C++ SDK dependancies
21  [here](https://github.com/aws/aws-sdk-cpp/blob/master/third-party/CMakeLists.txt#L18)). If
22 you install aws-c-common before building the AWS SDK for C++, this will fool the AWS SDK for
23 C++ install logic, and you will be forced to install several other dependencies manually. Version 1.7.231 of the
24 AWS SDK for C++ is supported by version v1.0.1 of the AWS Encryption SDK for C.
25 
26 You need to compile both the AWS Encryption SDK for C and its dependencies as either all
27 shared or all static libraries. We will use all shared library builds in our examples by
28 using the cmake argument `-DBUILD_SHARED_LIBS=ON`. You can change them to static library
29 builds by just changing `ON` to `OFF`.
30 
31 Once you have built each dependency, install it so it can be picked up by the next build.
32 
33 ### If you are working on an EC2 instance, regardless of operating system
34 
35 For best results, create an EC2 instance with an instance profile that at a
36 minimum has KMS permissions for Encrypt, Decrypt, and GenerateDataKey for
37 at least one KMS CMK in your account. You will not need any other AWS
38 permissions to use the AWS Encryption SDK for C.

40 ## Build recipes
41 
42 We will demonstrate some simple build recipes for Linux, Mac, and Windows operating systems.
43 
44 The Linux and Mac recipes install everything in the standard directories in /usr/local. The
45 Windows recipe installs everything in an install directory placed at the directory you are in
46 when you start the build process. To change the installation directory, if desired, see the Tips
47 and Tricks section at the end of this README.
48 
49 You can do (Option 1) a C and C++ build, which will include integration with KMS, or you can do
50 (Option 2) a C only build, which will not include integration with KMS. In places where
51 the recipes diverge, these will be labeled as (Option 1) and (Option 2). Follow one of the two
52 options, but not both, depending on which installation you want to do.
53 
54 ### Building on Amazon Linux
55 
56 This recipe should work with a brand new Amazon Linux instance. Start in the directory where
57 you want to do your build.
58 
59 #### Amazon Linux: (Option 1) C and C++ build dependencies
60 
61 Run the following:
62 
63  sudo yum update
64  sudo yum install -y openssl-devel git gcc-c++ libcurl-devel
65 
66 The yum repo has an old version of CMake, so download CMake 3.9 or later from [their
67 website](https://cmake.org/) and make sure cmake is in your path.
68 
69 Both aws-sdk-cpp and aws-c-common are required, but the installation of aws-sdk-cpp will install
70 aws-c-common for you.
71 
72 Do a KMS-only build of the AWS SDK for C++. If you want to use the AWS SDK for C++ for
73 other AWS services, you can omit the `-DBUILD_ONLY="kms"` argument, but the build will take much longer.
74 
75  git clone https://github.com/aws/aws-sdk-cpp.git
76  mkdir build-aws-sdk-cpp && cd build-aws-sdk-cpp
77  cmake -DBUILD_SHARED_LIBS=ON -DBUILD_ONLY="kms" -DENABLE_UNITY_BUILD=ON ../aws-sdk-cpp
78  make && sudo make install ; cd ..
79 
80 Now skip to the "Amazon Linux: Build and install the AWS Encryption SDK for C" section below.
81 
82 #### Amazon Linux: (Option 2) C only build dependencies
83 
84 Run the following:
85 
86  sudo yum update
87  sudo yum install -y openssl-devel git gcc
88 
89 The yum repo has an old version of CMake, so download CMake 3.9 or later from [their
90 website](https://cmake.org/) and make sure cmake is in your path.
91 
92 Now build and install aws-c-common:
93 
94  git clone https://github.com/awslabs/aws-c-common.git
95  mkdir build-aws-c-common && cd build-aws-c-common
96  cmake -DBUILD_SHARED_LIBS=ON ../aws-c-common
97  make && sudo make install ; cd ..

261 ## Compiling your program using the AWS Encryption SDK for C
262 
263 Once you have installed the AWS Encryption SDK for C, you are ready to start writing
264 your own programs with it.
265 
266 When doing a C compilation (not using the KMS keyring) be sure to include the flags
267 ``-lcrypto -laws-encryption-sdk -laws-c-common``.
268 
269 When doing a C++ compilation (using the KMS keyring) be sure to include the flags
270 ``-std=c++11 -lcrypto -laws-encryption-sdk -laws-encryption-sdk-cpp -laws-c-common -laws-cpp-sdk-kms -laws-cpp-sdk-core``.
271 
272 In the examples directory of this repo are several self-standing C and C++ files.
273 On a successful build, these files will already be compiled in the examples subdirectory
274 of the build directory. Additionally, if you want a quick test that you have
275 built and installed the AWS Encryption SDK successfully, you can copy any of
276 the example files out of that directory and build them yourself.
277 
278 Here are sample command lines using gcc/g++ to build a couple of the example files,
279 assuming that the libraries and headers have been installed where your system
280 knows how to find them.
281 
282  g++ -o string string.cpp -std=c++11 -lcrypto -laws-encryption-sdk -laws-encryption-sdk-cpp -laws-c-common -laws-cpp-sdk-kms -laws-cpp-sdk-core
283  gcc -o raw_aes_keyring raw_aes_keyring.c -lcrypto -laws-encryption-sdk -laws-c-common
284 
285 Note that the C++ files using the KMS keyring will require
286 you to make sure that AWS credentials are set up on your machine to run properly.

288 ## Tips and tricks
289 
290 When building aws-sdk-cpp, you can save time by only building the subcomponents needed
291 by the AWS Encryption SDK for C with `-DBUILD_ONLY="kms"`
292 
293 The `-DENABLE_UNITY_BUILD=ON` option will further speed up the aws-sdk-cpp build.
294 
295 To enable debug symbols, set `-DCMAKE_BUILD_TYPE=Debug` at initial cmake time,
296 or use `ccmake .` to update the configuration after the fact.
297 
298 If desired, you can set the installations to happen in an arbitrary
299 directory with `-DCMAKE_INSTALL_PREFIX=[path to install to]` as an argument to cmake.
300 
301 If you set `CMAKE_INSTALL_PREFIX` for the dependencies, when building the AWS
302 Encryption SDK for C you must either (1) set `CMAKE_INSTALL_PREFIX` to the same path,
303 which will cause it to pick up the dependencies and install to the same directory
304 or (2) set `CMAKE_PREFIX_PATH` to include the same path, which will cause it to pick
305 up the dependencies but NOT install to the same directory.
306 
307 You can also use `-DOPENSSL_ROOT_DIR=[path to where openssl was installed]` to make
308 the build use a particular installation of OpenSSL.
309 
310 By default the cmake for this project detects whether you have aws-sdk-cpp-core
311 and aws-sdk-cpp-kms installed in a place it can find, and only if so will it build
312 the C++ components, (enabling use of the KMS keyring.) You can override the detection
313 logic by setting `-DBUILD_AWS_ENC_SDK_CPP=OFF` to never build the C++ components or
314 by setting `-DBUILD_AWS_ENC_SDK_CPP=ON` to require building the C++ components (and
315 fail if the C++ dependencies are not found.)
